تقرير سير العمل التقني

شرح متكامل وشامل لعمليات الـ API من الاستكشاف إلى الحجز والدفع النهائي

1

الاستكشاف المبدئي (Discovery)

تبدأ تجربة المستخدم عند فتح التطبيق، حيث يقوم بطلب البيانات الأساسية لبناء الواجهة الرئيسية واستكشاف الوجهات المتاحة.

GET /api/v1/banners GET /api/v1/locations GET /api/v1/countries

عناصر الاستكشاف:

  • اللافتات (Banners): جلب صور العروض والرحلات المميزة.
  • المواقع (Locations): جلب هيكلية الدول مع المدن التابعة لها لتسهيل البحث.
  • الأسئلة الشائعة (FAQs): توفير مساعدة فورية للمستخدم.
2

تصفح وبحث الرحلات (Trips & Search)

يتم عرض باقات الرحلات السياحية مع ميزات البحث المتقدم والتصفية بناءً على الوجهة أو السعر.

GET /api/v1/trips GET /api/v1/trips/{id}
{ "error": false, "data": { "id": 1, "title": "رحلة جزر المالديف الساحرة", "price": 5000, "base_capacity": 2, "extra_passenger_price": 1000, "image_url": "https://..." } }

الميزات الذكية:

التسعير الديناميكي: يتم حساب السعر النهائي تلقائياً بناءً على عدد المسافرين، السعة الأساسية، وسعر الفرد الإضافي.

نظام المفضلة: يمكن للمستخدم إضافة الرحلة لمفضلته للرجوع لها لاحقاً.

3

إنشاء الحجز (Seamless Booking)

بعد اختيار الرحلة، يقوم المستخدم بإدخال بيانات المرافقين. يضمن الـ API صحة البيانات وتوفر التذاكر المتبقية.

POST /api/v1/trips/book
{ "trip_id": 1, "tickets_count": 2, "passengers": [ { "name": "أحمد علي", "passport_number": "A123..." }, { "name": "سارة محمد", "passport_number": "B456..." } ] }

حالة الحجز: PENDING

يتم إنشاء الحجز بحالة "معلق" حتى يتم التأكد من إتمام عملية الدفع.

التحقق من التوفر

يتم فحص عدد التذاكر المتاحة في نفس اللحظة لمنع الحجز المزدوج.

4

دورة الدفع المتكاملة (Payment Lifecycle)

هنا تكمن القوة التقنية، حيث يتم التعامل مع 3 بوابات دفع مختلفة في واجهة واحدة موحدة.

POST /api/payment/initiate POST /api/payment/verify
اختيار وسيلة الدفع
بدء العملية (Initiate)
التوجيه لبوابة الدفع
الرجوع للتطبيق (Callback)
التحقق النهائي (Verify)

الوسائل المدعومة:

  • HyperPay: تدعم (مدى، فيزا، ماستركارد، أبل باي).
  • Tabby: تدعم التقسيط المريح.
  • Tamara: تدعم التقسيط وشراء الآن والدفع لاحقاً.
5

التأكيد النهائي والفواتير (Confirmation)

بمجرد التأكد من نجاح العملية من خلال الـ API، يقوم النظام بسلسلة من العمليات الآلية في الخلفية لضمان الدقة.

العمليات الآلية المنفذة:

  • تحديث حالة الحجز: تحول تلقائي من PENDING إلى CONFIRMED.
  • خصم المخزون: خصم عدد التذاكر من سجل الرحلة (Inventory Management).
  • توليد الفاتورة: إنشاء فاتورة إلكترونية PDF احترافية تدعم اللغة العربية.
  • تسجيل المدفوعات: توثيق العملية في جدول المدفوعات مع تفصيل نوع البطاقة.
{ "error": false, "message": "Payment successful and booking confirmed.", "data": { "status": "paid", "invoice_url": "https://.../storage/invoices/inv_1.pdf", "transaction_id": "T789123" } }
