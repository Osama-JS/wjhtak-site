Fare Rules
Fare Rules method is used to request the rules and conditions of a particular fare result per unique fare-basis code retrieved from the flight availability search. It contains the fare basis code and the relevant details associated with it, such as restrictions and penalties.

Fare Rule Request

Method	URL
POST	https://travelnext.works/api/aeroVE5/fare_rules
Request Parameters

Parameter	Data Type	Required	Description	Example
session_id	string	Yes	Session_id is a token granted on flight availablity search and is available in the success Availability search response.	7906efba-09db-4481-8c60-0d7f5b5e6c44
fare_source_code	string	Yes	Unique fare/flight Option identification code	MTY2ODE2Njg2Ml8yNjA5Mzk
fare_source_code_inbound	string	No	Unique fare/flight Option identification code of Inbound segment, from Indian Domestic RoundTrip Search.	7906efba-09db-4481-8c60-0d7f5b5e6c44
Response Parameters

#	Parameter	Data Type	Required	Description	Example
1	BaggageInfos	Nested Array	Yes
1.1	BaggageInfo	Sub Array	Yes	Information of the baggage.
1.1.1	Arrival	string	Yes	Three letter IATA code which specifies the arrival airport code	DXB
1.1.2	Baggage	string	Yes	It specifies the baggage quantity. Sample: 1P, 30K. P-Piece, KKilogram and SB-Standard Baggage	1P
1.1.3	Departure	string	Yes	Three letter IATA code whichspecifies the departure airport code.	DEL
1.1.4	Flight No	string	Yes	Operating flight Number.	BA729
2	FareRules	Nested Array	Yes
2.1	FareRule	Nested Array	Yes
2.1.1	Airline	string	Yes	It specifies the operating airline Code	BA
2.1.2	CityPair	string		It specifies the origin and destination airport codes	DELDXB
2.1.3	category	string	No	It specifies the category of the rules and regulation	Day / Time
2.1.4	Rules	string	No	It returns a detailed description of the rules under the specified category	NO DAY/TIME TRAVEL RESTRICTIONS APPLY.
Important Notes

Indian Domestic RoundTrip - In case of Indian domestic round trip, the Outbound & Inbound fare rules details will be available separately. The Outbound results will be available in the FareRules1_1Result parameter and the Inbound results will be available in the FareRules1_1ResultInbound parameter.
